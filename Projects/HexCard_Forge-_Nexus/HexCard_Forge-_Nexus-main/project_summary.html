HexCard Forge Nexus

Hybrid Desktop & Web Toolkit for Card Forging Strategy Games

Project Overview

HexCard Forge Nexus is a unified application (web & desktop) for creating and playing a hex-card strategy game. It combines three core features:

  1. Forge Characters: Fuse six hex-cards to create unique characters with combined stats.
  2. Map Editor: Design or procedurally generate hexagonal maps for gameplay.
  3. Card Game (Play Mode): Play a turn-based strategy game where forged characters battle on the hex map.

All workflows share a common data model and UI theme, allowing seamless switching between Forge, Map, and Play screens. The app runs both as a Tauri-powered desktop app and a browser client, with shared logic and components.

Key Features

  • Seamless switching between Forge, Map, and Play screens (no reloads).
  • Every hex-card is usable anywhere (no mode-specific locks).
  • Map editor supports manual and procedural generation with seeds.
  • Save/load entire project (maps, decks, forged characters, game state) to disk or browser storage.
  • AI assistants for card combos and map generation (planned).
  • Accessible (keyboard navigation, color-blind friendly, WCAG 2.2 AA).
  • Runs offline (desktop) and online (browser).

Core Data Models

  • HexCard: { id, name, type, edges, tags, cost, text } – six-edged card with icons and stats.
  • Character: { id, name, cards[], stats, abilities[] } – forged from six cards.
  • MapTile: { id, terrain, occupant } – hex grid tile.
  • SaveGame: { map, deck, forged[], turnState, ... } – full game state.

All data validated with Zod schemas for runtime safety.

Development Workflow

  • Test-driven, prompt-based: Each feature is built via an LLM prompt and validated by automated tests before moving on.
  • Global state via Zustand, persisted to localStorage (web) or disk (desktop).
  • Continuous Integration: GitHub Actions for builds and tests (web, Windows, macOS, Linux).

How to Run

  1. Install dependencies: pnpm install
  2. Start desktop app: pnpm dev:desktop
  3. Start web client: pnpm dev:web
  4. Build desktop app: pnpm --filter desktop-studio tauri build

Game Mechanics

HexCard Forge's gameplay revolves around a unique fusion system:

HEX CARD
Six-sided with unique edges

Forge Mechanics

  • Six cards arranged in a hexagonal pattern combine to create one character
  • Each edge contributes to the character's stats based on its position
  • Edge types: attack, defense, element, resource, skill, and link
  • Strategic combinations unlock synergy bonuses and special abilities

Battle Mechanics

  • Turn-based strategy on a hexagonal grid
  • Characters move and attack based on their forged stats
  • Terrain types affect movement and abilities
  • Dynamic line-of-sight and range calculations
  • Weather and environmental effects modify gameplay

TDD Development Plan

The project follows a unique test-driven development approach powered by AI:

  1. Prompt-based chunks: Development is broken into manageable chunks with AI prompts
  2. Test-first implementation: Each prompt includes tests that must pass before proceeding
  3. Red-Green-Refactor: Start with failing tests, implement code to make them pass, then refactor
  4. Incremental milestones: Eight major milestones (M-1 through M-8) track progress
"Each prompt describes a set of tasks and the tests that must pass, and the LLM returns a patch implementing those tasks. The developer then runs the tests; only if they all pass ('green') do they proceed to the next prompt."
